# Лабораторная работа №3: Очередь задач и генераторы
**Тема:** Реализация протоколов итерации и генераторов в Python

## Описание проекта
В рамках работы разработана кастомная коллекция `TaskQueue`, предназначенная для управления потоком задач. Ключевой фокус сделан на оптимизации потребления памяти и поддержке ленивых вычислений. Коллекция реализует полный протокол итерации Python, что гарантирует нативную интеграцию со встроенными конструкциями языка: циклами `for`, приведением к `list()` и агрегацией через `sum()`.

---

## Архитектура компонентов
Архитектура проекта разделена на три логических модуля, строго типизированных с использованием аннотаций:

* **`Task` (Сущность задачи)**  
  Хранит метаданные отдельной задачи.  
  ▸ **Поля:** `name`, `priority`, `status`, `duration`.  
  ▸ **Особенность:** Переопределён метод `__radd__`, что позволяет функции `sum()` корректно суммировать объекты задач с целочисленным аккумулятором для расчёта общего времени.

* **`TaskIterator` (Механизм обхода)**  
  Отвечает за последовательную итерацию и управление состоянием.  
  ▸ Хранит текущий индекс элемента.  
  ▸ Реализует стандартные дандер-методы `__iter__` и `__next__`.  
  ▸ Корректно сигнализирует об окончании обхода через возбуждение `StopIteration`.

* **`TaskQueue` (Контейнер)**  
  Итерируемая коллекция, управляющая внутренним списком задач.  
  ▸ Метод `__iter__` при каждом вызове возвращает **изолированный** экземпляр `TaskIterator`, что позволяет безопасно обходить очередь многократно.  
  ▸ Включает методы потоковой фильтрации по статусу и приоритету.

---

## Технические решения
В соответствии с требованиями проекта реализованы следующие оптимизации:

* **Нативный протокол итерации**  
  Разделение логики контейнера и состояния обхода позволяет бесшовно работать со стандартными функциями вроде `list()`, не требуя явных методов конвертации или копирования данных.

* **Ленивая обработка через генераторы**  
  Фильтрация реализована с использованием ключевого слова `yield`. Это исключает создание промежуточных списков в памяти: элементы обрабатываются и отдаются вызывающему коду строго по мере необходимости, что обеспечивает стабильную работу даже при больших объёмах данных.

* **Современная статическая типизация**  
  Используется `from __future__ import annotations`, что упрощает чтение сигнатур, избегает проблем с прямой ссылкой на ещё не определённые классы и делает архитектуру самодокументируемой без дополнительных зависимостей.

## Пример использования

**Добавление задач, агрегация времени и потоковая фильтрация:**
```python
queue = TaskQueue()
queue.add_task(Task("Аналитика", priority=1, status="todo", duration=4))

# Использование встроенной функции sum()
total_time = sum(queue) 
print(f"Общее время выполнения: {total_time} ч.")

# Потоковая обработка без загрузки всего списка в RAM
for task in queue.filter_by_priority(2):
    print(task.name)
```
